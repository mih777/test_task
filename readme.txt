Сервер всегда работает в многопоточном режиме, а не только когда большая нагрузка.

Работает следующим образом:

- клиент генертрует случайное количество обьектов с заданной структурой и отправляет

все это дело на сервер

- количество обьектов и частота отправки можно задать в клиенте ( index.html )

- сервер принимает отправленные данные , парсит обьекты и выводит в консоль, 

логирует в файл log.txt, записывает обьекты в файл objects.json ( последнюю отправку.

можно сделать чтобы все сохранял, просто файл огромный получитсяя, но если нужно то не 
проблема )

- Сервер сам себя перезагружает, в случае если какой то процесс умрет, то есть
перезапускает процессы, не падает. Отказоустойчивость.

=====================================================================

Что бы запустить приложение:

1. открываем папку с приложением в консоле и запускаем командой "npm start" , 
НЕ ЗАБЫВ СДЕЛАТЬ "npm install", чтобы установить зависимости.


это запустится сервер. Запустится на всех ядрах процессора, сколько бы их не было.

2. в корне есть файл index.html, его открываем в браузере. Это клиент.

там две кнопки: "Начать отправку", "Остановить отправку". На экран выводятся кол-во

отправленных обьектов. Если нажать кнопку "Остановить отправку" , отправка прекратится

и выведется сообщение "data sending stoped"
